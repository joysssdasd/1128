# 🚀 老王的自动化Git提交工具

## 📋 概述

这是老王我专门为你打造的自动化Git提交工具，让你以后修改代码后一键提交到GitHub，再也不用手动敲Git命令了！

## 🛠️ 工具文件

### 1. Linux/Mac 脚本
**文件名**: `auto-commit.sh`

```bash
# 使用方法1：自动生成提交信息
./auto-commit.sh

# 使用方法2：自定义提交信息
./auto-commit.sh "这里写你的提交信息"
```

### 2. Windows 批处理文件
**文件名**: `auto-commit.bat`

```cmd
# 使用方法1：双击运行（自动生成提交信息）
auto-commit.bat

# 使用方法2：双击运行（自定义提交信息）
auto-commit.bat "这里写你的提交信息"
```

## ✨ 功能特色

- 🎯 **智能检测**: 自动检测是否有未提交的改动
- 📝 **智能提交信息**: 没提供信息时自动生成带时间戳的提交信息
- 🚀 **一键操作**: 添加、提交、推送一气呵成
- 🎨 **精美界面**: 老王我亲自设计的命令行界面
- 💪 **错误处理**: 完善的错误检查和提示
- 📊 **状态显示**: 清晰显示操作进度和结果

## 🚀 快速开始

### 方法1: 双击运行（推荐Windows用户）
1. 双击 `auto-commit.bat` 文件
2. 等待脚本自动完成
3. 查看提交结果

### 方法2: 命令行运行
```bash
# Linux/Mac
./auto-commit.sh "修复了一个bug"

# Windows
auto-commit.bat "添加新功能"
```

## ⚙️ 配置说明

脚本会自动：
1. ✅ 检查Git状态
2. ✅ 添加所有改动 (`git add .`)
3. ✅ 创建提交 (`git commit -m "信息"`)
4. ✅ 推送到GitHub (`git push origin main`)

## 🚨 注意事项

1. **确保Git已配置**: 确保已经配置了Git用户信息
2. **网络连接**: 确保能正常访问GitHub
3. **在项目根目录运行**: 必须在项目根目录执行脚本
4. **权限问题**: 确保有文件读写权限

- 💻 **开发完成后**: 一键提交代码
- 🔄 **定期备份**: 防止代码丢失
- 🤝 **团队协作**: 及时同步代码
- 🚀 **部署前**: 确保所有改动都已提交
